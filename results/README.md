# Results Directory

`results/` contains tracked experiment summaries. Prefer JSON for structured
records and CSV only when a table is the primary output.

Every new result should include, when practical:

- experiment name;
- config and seeds;
- agent names and versions;
- wall-clock time;
- summary statistics;
- source script path;
- git commit or a note that the tree was dirty.

Raw corpora, checkpoints, and huge intermediate arrays belong in `corpora/`,
`data/`, `artifacts/`, or `checkpoints/`, with a summary result committed here.
